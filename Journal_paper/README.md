# How are HPC, Edge and Serverless Architectures Built in the Cloud Continuum?

## An Analysis of Real Cloud Architectures in AWS

Reproducibility package for the paper *“How are HPC, Edge and Serverless Architectures Built in the Cloud Continuum? An Analysis of Real Cloud Architectures in AWS”* by Steven Santillán and [Cristina L. Abad](https://sites.google.com/fiec.espol.edu.ec/cv-cabad/english) from [ESPOL University](https://www.espol.edu.ec/en).

This package contains the data, source code, analysis scripts, and generated figures required to reproduce the analyses presented in the journal paper. The study examines how real-world AWS cloud architectures are built across the cloud continuum, with particular focus on High-Performance Computing (HPC), Edge, Serverless, and Function-as-a-Service (FaaS) architectures.

## Reproducibility Scripts

The main reproducibility notebook is available at:

[**`hpc_n_edge_clouds_archs_extends_IJNDC_VF_UP.ipynb`**](./hpc_n_edge_clouds_archs_extends_IJNDC_VF_UP.ipynb)

## Package Contents

This reproducibility package includes:

* The dataset of AWS cloud architectures used in the study.
* Architecture metadata and analytical-category labels.
* Scripts and notebooks for data preparation and classification.
* Analyses of service usage across HPC, Edge, Serverless, FaaS, and related architectural categories.
* Generated figures and tables reported in the paper.
* Supporting files required to reproduce the experimental results.

## How to Reproduce the Analysis

1. Clone this repository.
2. Navigate to the `journal_paper` directory.
3. Install the Python dependencies required by the notebook or script.
4. Open and execute `hpc_n_edge_clouds_archs_extends_IJNDC_VF_UP.ipynb` sequentially.
5. The generated figures, tables, and intermediate datasets will be saved in the corresponding output directories.
